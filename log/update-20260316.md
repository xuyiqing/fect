Two-tier CV tolerance (`cv_tol = max(tol, 1e-3)`) for CV speed. Result: tol=1e-3 now gives 43-2249x better component accuracy with zero speed regression.

## Design Decisions

1. **Shared scoring as separate file**: `.score_residuals()` in `R/score.R` rather than inlined, making the dependency explicit.

2. **cv.method replaces cv.treat + mask.method**: Single string parameter (`"all_units"`, `"treated_units"`, `"loo"`) is more expressive than a boolean whose meaning varies by function.

3. **fect_mspe simplified to cv.sample-only**: Removed random/pre.trend/user masking — cv.sample with structured k-fold is strictly better. Breaking change accepted on cfe branch.

4. **1% selection rule hardcoded**: Replaces `tol`-based threshold (which conflated convergence tolerance with model selection) with fixed 0.01 matching `fect_cv`.

5. **Always inter_fe_ub for fold-level CV**: Masking creates an effectively unbalanced panel, so `inter_fe` (balanced-only) cannot be used even on originally balanced data.

6. **Factor estimation once per r for "treated_units"**: Factors estimated on full control panel per candidate r; only loadings re-estimated per fold. Correct and efficient.

7. **Joint ife() fallback in cfe_iter**: When no CFE components are present, `complex_fe_ub` delegates to `ife()` for joint FE+factor estimation, achieving exact equivalence with `inter_fe_ub`. Overhead: 2-7% on typical panels, acceptable.

8. **R centering for convergence**: Subtracting grand mean before C++ solver ensures `tol` measures variation precision, not level precision. Critical for data with large means (e.g., GDP).
